# 智能考研系统用户操作手册

## 1. 系统简介

智能考研系统用于个人考研学习资料整理、知识库问答、错题复盘和学习计划管理。用户可以上传资料，校正抽取文本，将资料加入知识库后围绕资料提问；也可以维护错题集、安排周学习计划，并配置自己的 AI 模型接口。

## 2. 启动系统

### 2.1 启动后端

在项目根目录进入后端目录：

```powershell
cd backend
mvn spring-boot:run
```

默认后端地址：

- API：`http://localhost:8080/api`
- H2 Console：`http://localhost:8080/h2-console`

### 2.2 启动前端

在项目根目录进入前端目录：

```powershell
cd frontend
npm.cmd install
npm.cmd run dev
```

浏览器打开 Vite 输出的地址。若后端地址不是默认值，可以设置：

```powershell
$env:VITE_API_BASE="http://localhost:8080/api"
npm.cmd run dev
```

### 2.3 可选环境

- 图片 OCR 需要安装 Tesseract，并确保 `tesseract` 命令可用。
- Elasticsearch 是知识库检索增强服务；不启动时系统仍可使用数据库本地检索兜底。
- AI 问答和 AI 规划需要在“AI 设置”中填写模型 Endpoint 和 API Key；不填写时知识库问答只能使用有限的本地摘要兜底。

## 3. 注册与登录

1. 打开前端页面。
2. 在登录/注册面板选择“注册”。
3. 输入用户名、密码和可选昵称。
4. 点击创建账号。
5. 已有账号时选择“登录”，输入用户名和密码后进入学习空间。
6. 点击侧边栏顶部退出按钮可退出登录。

### 3.1 首次学习信息初始化

新注册账号首次进入系统时，需要填写学科数量、各学科名称和考研日期。提交后系统会自动创建对应的一级学科文件夹，这些一级文件夹会作为后续学科画像、错题科目标签和知识片段统计的基础。

旧账号如果已经存在一级文件夹，系统会自动把这些一级文件夹标记为学科文件夹；如果没有一级文件夹，则会进入同样的初始化流程。

## 4. 我的知识库

“我的知识库”包含三个入口：我的资料、知识问答、上传编辑。

### 4.1 创建资料文件夹

1. 进入“我的知识库”。
2. 点击“我的资料”。
3. 在创建文件夹区域输入文件夹名称和可选描述。
4. 点击“创建”。
5. 如需创建子文件夹，先选择一个已有文件夹，再输入名称创建。

说明：当前前端限制文件夹最大深度为 3 级。

### 4.2 修改或删除文件夹

1. 在“我的资料”中找到文件夹卡片。
2. 点击编辑按钮修改名称和描述。
3. 点击删除按钮删除文件夹。

删除前请确认该文件夹下资料是否仍需保留。

### 4.3 上传资料

1. 先在侧边栏或“我的资料”中选择目标文件夹。
2. 进入“上传编辑”。
3. 选择资料标签：教材、资料、笔记、习题或其他。
4. 选择本地文件并上传。
5. 上传后系统会自动抽取文本，并显示在编辑区。

支持的常见资料包括 PDF、Word、图片、文本和 Markdown。图片识别依赖本机 OCR。

### 4.4 编辑抽取文本

1. 上传完成后查看编辑区文本。
2. 如果 OCR 或文档解析不准确，可以直接手动修改。
3. 点击保存，将文本重新保存并重建知识片段。

### 4.5 加入或移出知识库

在“我的资料”文件列表中，每个文件会显示是否已加入知识库：

- 点击“移出”可让该文件不再参与知识问答。
- 点击“加入”可重新加入知识库。

加入知识库后，系统会把文本切分成片段，用于后续检索问答。

### 4.6 移动或删除文件

在文件列表中：

- 点击移动按钮，选择目标文件夹后提交。
- 点击删除按钮，确认后删除文件。

删除文件会同步删除其知识片段和本地文件。

## 5. 知识问答

### 5.1 基于资料提问

1. 在侧边栏选择一个文件夹。
2. 进入“知识问答”。
3. 保持“使用知识库”开启。
4. 输入问题并发送。
5. 系统会检索当前文件夹及其子文件夹中的知识片段，并生成答案。

### 5.2 查看来源引用

开启“引用来源”后，答案中会出现 `[1]`、`[2]` 等编号。点击编号可以查看对应资料片段、掌握度和引用次数。每个来源还提供“很清楚”和“忘记了”按钮，用于把本次反馈回写到该知识片段的正确或错误命中次数中。需要进一步核对时，可打开来源文件并定位到相关文本。

### 5.3 深度回答

开启“深度回答”后，系统会额外进行查询改写和补充检索，适合综合性、跨章节或表述较口语化的问题。该模式响应时间会更长。

### 5.4 教师模式

将问答模式切换为教师模式后，系统固定使用知识库和知识溯源，每次只生成一个问题。用户可以在输入框中填写提问要求，例如“CPU”或“进程与线程”，也可以选择学科范围。生成问题后，可以点击“很清楚”“忘记了”记录反馈，点击“添加到错题”把该题和来源知识片段一起保存到错题集，点击“下一题”继续生成同一要求下的其他问题。

教师模式会综合相关性、掌握度、遗忘风险和引用次数选择知识片段。若资料中没有明确支撑，系统会尽量生成更保守的问题，避免要求用户回答资料没有直接覆盖的内容。

### 5.5 直接聊天

关闭“使用知识库”后，系统不检索资料，不返回来源引用，直接调用用户配置的聊天模型。此模式需要在“AI 设置”中配置有效的聊天模型 API Key。

### 5.6 会话生成笔记

在当前文件夹下完成一轮问答后，可以使用生成笔记功能，将对话整理成一份文本笔记并保存到当前文件夹。

### 5.7 知识画像

“知识画像”页面展示个人学习统计，包括整体掌握度、覆盖率、薄弱点数、待复习数量和距离考研天数。页面还提供学科画像、教材画像和薄弱知识点列表。

薄弱知识点可以直接进入教师模式复习。系统只把已经发生过反馈的知识片段纳入覆盖率和薄弱知识统计，避免新导入但尚未学习的资料影响判断。

## 6. 学习规划

学习规划包含“自我规划”和“AI 规划”。

### 6.1 自我规划

1. 进入“学习规划”。
2. 选择“自我规划”。
3. 使用上周、本周、下周按钮切换周视图。
4. 点击某一天新增计划。
5. 填写标题、科目、类型、日期、开始时间、结束时间、地点、优先级、状态和说明。
6. 保存后计划会显示在周课表中。

计划类型包括课程、自习、复盘、考试、任务、休息。计划状态包括待办、完成、跳过。

### 6.2 修改和删除计划

1. 在周课表中点击某个计划块。
2. 在右侧表单修改内容后保存。
3. 如需删除，使用删除按钮。
4. 页面提供撤回按钮，可撤回部分最近修改操作。

### 6.3 AI 规划

1. 进入“学习规划”。
2. 选择“AI 规划”。
3. 与 AI 描述自己的目标、考试时间、每日可用时间和薄弱科目。
4. 生成规划草稿。
5. 查看预览和操作列表。
6. 确认无误后应用到真实日程。

注意：AI 生成的规划不会自动写入日程，必须用户确认后才会保存。

## 7. 错题集

错题集包含“上传错题”“随机练习”“浏览错题”等功能。

### 7.1 上传错题

1. 进入“错题集”。
2. 选择“上传错题”。
3. 填写题干文本。
4. 可上传题目文件或题目图片。
5. 填写解析文本。
6. 可上传解析文件或解析图片。
7. 选择是否完全掌握。
8. 选择掌握状态和科目标签。
9. 在“关联知识片段”区域选择学科文件夹、资料文件并搜索关键词，点击候选知识片段进行关联。
10. 保存错题。

如果暂时不关联知识片段，错题仍然可以保存，但后续刷题结果不会回写知识画像。

### 7.2 文件识别

上传错题时，如果有题目文件或图片，可以使用识别功能提取文本。识别结果可以继续手动修改。

### 7.3 掌握状态管理

系统内置“未掌握”和“完全掌握”语义。用户还可以创建自定义状态，例如“需要二刷”“公式不熟”“已订正”。如果状态已被错题使用，系统会阻止删除。

### 7.4 科目标签管理

系统会把一级学科文件夹自动同步为同名科目标签。用户也可以继续创建其他补充标签，例如“公式不熟”“概念混淆”。错题可以绑定多个标签。如果标签已被使用，系统会阻止删除。

### 7.5 随机练习

1. 进入“错题集”。
2. 选择“随机练习”。
3. 设置抽题数量。
4. 可选择科目标签作为筛选条件。
5. 开始练习。

系统会从未完全掌握的错题中抽取题目。每道练习题可以点击“写对了”或“写错了”记录本轮结果：如果该错题关联了知识片段，系统会同步更新相关 chunk 的正确或错误命中次数，并刷新最近练习时间；如果没有关联知识片段，则只记录本轮操作提示，不回写知识画像。

### 7.6 浏览错题

在浏览错题中可以查看错题、题目附件、解析附件、掌握状态和标签。可根据界面提供的筛选项查看未掌握或已掌握题目。

## 8. AI 设置

进入“AI 设置”后可以配置：

| 配置项 | 说明 |
| --- | --- |
| AI 角色 | 例如“严谨的考研答疑老师” |
| 系统提示词 | 控制回答风格和依据 |
| 聊天模型 | 如 `gpt-4o-mini` 或其他兼容模型名 |
| 聊天 Endpoint | OpenAI 兼容 `/v1/chat/completions` 地址 |
| 聊天 API Key | 用户自己的模型 Key |
| Embedding 模型 | 如 `text-embedding-3-small` |
| Embedding Endpoint | OpenAI 兼容 `/v1/embeddings` 地址 |
| Embedding API Key | 用于向量检索的 Key |
| Embedding 维度 | 默认 1536 |

保存后，后端会按当前用户保存配置。前端还支持本地预设，方便切换不同模型配置。

## 9. 常见问题

### 9.1 上传图片后没有识别出文字

请确认本机安装了 Tesseract，并且 `application.yml` 中 `app.ocr.tesseract-command` 指向可用命令。未安装 OCR 时，可以手动在编辑区补充文本。

### 9.2 问答没有引用来源

请检查：

1. 是否开启“使用知识库”。
2. 是否选择了文件夹。
3. 资料是否已加入知识库。
4. 是否开启“引用来源”。

### 9.3 直接聊天不可用

直接聊天不使用本地知识库，需要有效的聊天模型 API Key、模型名和 Endpoint。

### 9.4 Elasticsearch 没启动是否影响使用

不影响基础使用。系统会回退到数据库本地检索，但语义检索效果会弱于开启 Elasticsearch 和 embedding 的情况。

### 9.5 AI 规划会不会直接改日程

不会。AI 规划先生成草稿和预览，用户确认应用后才会写入真实学习计划。

## 10. 数据备份建议

本系统默认把数据保存在：

- 数据库：`backend/data/`
- 上传文件：`backend/uploads/` 或配置的上传目录

备份时应同时备份数据库文件和上传目录，否则数据库记录与本地文件可能不一致。
