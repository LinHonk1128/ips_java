# 智能考研系统用户操作手册

## 1. 系统简介

智能考研系统用于个人考研备考管理。用户可以上传资料，校正抽取文本，将资料加入知识库后进行问答和定制练题；也可以查看知识画像、维护错题、随机练习、安排周学习计划，并配置自己的 AI 模型接口。

## 2. 启动系统

### 2.1 启动后端

在项目根目录进入后端目录：

```powershell
cd backend
mvn spring-boot:run
```

默认后端地址：

- API：`http://localhost:8080/api`

后端默认连接本机 MySQL 的 `smart_exam` 数据库。首次启动前请确认 MySQL 服务可用，并根据本机情况修改 `backend/src/main/resources/application.yml` 中的数据源账号、密码、JWT Secret 和模型配置。正式使用时不要把真实 API Key 提交到代码仓库。

如需加载演示数据，可使用 `demo` profile：

```powershell
mvn.cmd spring-boot:run "-Dspring-boot.run.profiles=demo"
```

### 2.2 启动前端

在项目根目录进入前端目录：

```powershell
cd frontend
npm.cmd install
npm.cmd run dev
```

浏览器打开 Vite 输出的地址。如果后端不是默认地址，可以设置：

```powershell
$env:VITE_API_BASE="http://localhost:8080/api"
npm.cmd run dev
```

### 2.3 可选环境

- 图片 OCR 需要安装 Tesseract，并保证 `tesseract` 命令可用。
- Elasticsearch 用于增强知识库检索，不启动时系统会回退数据库检索。
- AI 问答、AI 规划和 AI 诊断增强需要在“AI 设置”中填写兼容接口的模型名、Endpoint 和 API Key。

## 3. 注册、登录与初始化

1. 打开前端页面。
2. 选择“注册”，输入用户名、密码和可选昵称，创建账号。
3. 已有账号时选择“登录”，输入用户名和密码进入系统。
4. 首次进入系统时填写考研日期和学科名称。
5. 提交后系统会自动创建一级学科文件夹。
6. 点击侧边栏的退出入口可退出登录。

一级学科文件夹会作为知识画像、错题标签和学习统计的基础。后续可在“个人设置”中维护考研日期和学科。

## 4. 我的知识库

“我的知识库”包含“我的资料”“知识问答”“上传编辑”三个入口。

### 4.1 创建文件夹

1. 进入“我的知识库”。
2. 打开“我的资料”。
3. 选择根目录或某个已有文件夹。
4. 输入文件夹名称和可选描述。
5. 点击创建。

系统最多支持 3 级文件夹。一级文件夹通常对应考研学科，二级和三级可按章节、专题或资料来源划分。

### 4.2 修改或删除文件夹

在“我的资料”中选择文件夹后，可使用编辑入口修改名称和描述，也可以删除文件夹。删除前请确认其中资料和子文件夹是否仍需保留；若服务层检测到不允许删除，会返回提示。

### 4.3 上传资料

1. 先选择目标文件夹。
2. 进入“上传编辑”。
3. 选择标签：教材、资料、笔记、习题或其他。
4. 选择本地 PDF、Word、图片、TXT 或 Markdown 文件并上传。
5. 上传后系统会自动抽取文本并显示在编辑区。

图片识别依赖本机 OCR。如果识别结果不准确，可以在编辑区手动修改。

### 4.4 编辑抽取文本

1. 在编辑区检查抽取内容。
2. 修正乱码、漏识别、公式说明或段落结构。
3. 点击保存。

保存后系统会重新生成知识片段，用于后续问答和画像统计。

### 4.5 加入或移出知识库

文件默认会加入知识库。用户可以在文件列表中切换知识库状态：

- 移出知识库：删除该文件对应知识片段，不再参与问答。
- 加入知识库：重新根据抽取文本生成知识片段。

### 4.6 移动或删除文件

文件列表中可以移动文件到其他文件夹，也可以删除文件。删除文件会同步删除知识片段和本地上传文件。

## 5. 知识问答

### 5.1 基于资料提问

1. 在侧边栏选择一个文件夹。
2. 进入“知识问答”。
3. 保持“使用知识库”开启。
4. 输入问题并发送。
5. 系统会检索当前文件夹及子文件夹中的知识片段，再生成回答。

### 5.2 查看来源引用

开启“引用来源”后，答案会带有 `[1]`、`[2]` 等来源编号。点击来源可以查看资料名称、页码、片段摘录、引用次数、正确/错误反馈和掌握度。

每个来源可提交：

- “很清楚”：增加该片段的正确反馈。
- “忘记了”：增加该片段的错误反馈。

这些反馈会进入知识画像统计。

### 5.3 深度回答

开启“深度回答”后，系统会对问题进行额外改写和补充检索，适合跨章节、综合性或表述较模糊的问题。该模式会比普通问答更慢。

### 5.4 定制练题

1. 在知识问答页面切换为定制练题。
2. 选择知识库文件夹，可选具体学科范围。
3. 输入提问要求，例如“进程调度”“线性代数特征值”。
4. 系统生成一个练习题、参考答案和来源片段。
5. 可点击“很清楚”或“忘记了”记录反馈。
6. 可点击“加入错题”把该题保存到错题集并关联来源 chunk。
7. 点击下一题继续抽问。

定制练题会综合相关性、掌握度、遗忘风险和少量随机因素选择题目来源。

### 5.5 直接聊天

关闭“使用知识库”后，系统不会检索资料，也不会返回来源引用，而是直接调用聊天模型。此模式必须配置有效聊天模型 API Key。

### 5.6 会话生成笔记

完成一轮问答后，可以使用生成笔记功能，把当前会话整理成文本笔记并保存到当前文件夹。

## 6. 知识画像

知识画像页面根据资料片段的引用、反馈、错题练习和访问记录生成学习统计。

主要内容包括：

- 整体掌握度、覆盖率、薄弱点、高风险点和考研倒计时。
- 学科画像和资料画像。
- 薄弱知识点列表。
- 近期趋势、掌握度分布和学习活动。
- 高风险知识点、复习压力和诊断建议。

如果数据不足，系统会提示先进行问答反馈或错题练习。薄弱点和诊断建议可以进入定制练题，也可以加入学习计划。

## 7. 学习规划

学习规划包含“自我规划”和“AI 规划”。

### 7.1 自我规划

1. 进入“学习规划”。
2. 选择“自我规划”。
3. 使用上周、本周、下周切换周视图。
4. 点击日期或新增入口创建计划。
5. 填写标题、科目、类型、日期、开始时间、结束时间、地点、优先级、状态和说明。
6. 保存后计划显示在周课表中。

计划类型包括课程、自习、复盘、考试、任务、休息。计划状态包括待办、完成、跳过。

### 7.2 修改、删除和撤回

点击周课表中的计划块可编辑。保存后计划立即更新。删除入口可移除计划。页面提供撤回功能，可撤回最近的部分前端草稿修改。

### 7.3 AI 规划

1. 进入“学习规划”并选择“AI 规划”。
2. 描述目标、可用时间、薄弱科目和本周安排。
3. 与 AI 多轮讨论。
4. 生成规划草稿。
5. 检查操作列表和预览日程。
6. 确认无误后应用到真实日程。

AI 规划不会直接修改数据库，必须用户确认应用后才会保存。

## 8. 错题集

错题集包含“上传错题”“随机练习”“浏览错题”。

### 8.1 上传错题

1. 进入“错题集”。
2. 选择“上传错题”。
3. 填写题干文本，可上传题目文件或题目图片。
4. 填写解析文本，可上传解析文件或解析图片。
5. 设置是否完全掌握和自定义状态。
6. 选择科目标签。
7. 在“关联知识片段”区域按学科、资料和关键词搜索 chunk。
8. 选择相关片段后保存错题。

错题可以不关联知识片段，但后续练习结果不会回写知识画像。

### 8.2 文件识别

上传错题文件或图片后，可以使用识别功能抽取文本。识别结果可继续人工修改。

### 8.3 掌握状态管理

系统内置“未掌握”和“完全掌握”语义。用户还可以创建自定义状态，例如“需要二刷”“公式不熟”“概念混淆”。如果状态已被错题使用，系统会阻止删除。

### 8.4 科目标签管理

一级学科文件夹可同步为同名科目标签。用户也可以添加补充标签。错题可以绑定多个标签。如果标签已被使用，系统会阻止删除。

### 8.5 随机练习

1. 进入“随机练习”。
2. 设置抽题数量。
3. 可选择科目标签筛选。
4. 开始练习。
5. 每道题可点击“写对了”或“写错了”。

如果错题关联了知识片段，系统会把练习结果同步回写到相关 chunk 的正确或错误次数，并刷新最近练习时间。

### 8.6 浏览错题

“浏览错题”中可查看题干、解析、题目附件、解析附件、掌握状态、科目标签和关联知识片段，也可以编辑或删除错题。

## 9. AI 设置

进入“AI 设置”后可配置：

| 配置项 | 说明 |
| --- | --- |
| AI 角色 | 例如“严谨的考研答疑老师” |
| 系统提示词 | 控制回答依据和风格 |
| 聊天模型 | 兼容 Chat Completions 的模型名 |
| 聊天 Endpoint | 兼容 `/v1/chat/completions` 的地址 |
| 聊天 API Key | 用户自己的模型密钥 |
| Embedding 模型 | 兼容 Embeddings 的模型名 |
| Embedding Endpoint | 兼容 `/v1/embeddings` 的地址 |
| Embedding API Key | 用于向量检索的密钥 |
| Embedding 维度 | 默认 1536，可按模型调整 |

系统支持保存多个用户级预设，方便切换不同服务商或模型组合。

## 10. 常见问题

### 10.1 图片没有识别出文字

请确认已安装 Tesseract，并且 `application.yml` 中 `app.ocr.tesseract-command` 指向可用命令。未安装 OCR 时，可以手动补充文本。

### 10.2 问答没有来源

请检查是否开启“使用知识库”和“引用来源”，是否选择了文件夹，资料是否已加入知识库，且文件文本是否成功保存。

### 10.3 直接聊天不可用

直接聊天依赖聊天模型 API Key。请在“AI 设置”中配置模型名、Endpoint 和 Key，或重新开启知识库使用本地摘要兜底。

### 10.4 Elasticsearch 没启动是否影响使用

不影响基础流程。系统会回退数据库检索，但语义检索效果通常弱于开启 ES 和 Embedding 的情况。

### 10.5 AI 规划会不会自动改日程

不会。AI 规划先生成草稿和预览，用户确认应用后才会写入真实学习计划。

### 10.6 知识画像没有数据

知识画像依赖来源反馈和错题练习。上传资料后，请先进行问答、定制练题反馈或错题练习。

## 11. 数据备份建议

系统默认数据位置：

- 数据库：MySQL 数据库 `smart_exam`
- 上传文件：后端配置的 `uploads` 目录

备份时应同时导出 MySQL 数据库并备份上传目录，否则数据库记录和本地文件可能不一致。
