# 智能考研系统项目重点突出内容

## 1. 项目定位

本项目是一个面向个人考研学习场景的智能知识库与学习闭环系统。系统以用户上传的教材、笔记、习题、图片等资料为基础，完成文本抽取、语义切片、知识问答、来源反馈、知识画像、定制练题、错题复盘和学习计划联动，目标是把“资料管理工具”升级为“可诊断、可追踪、可复习”的个人学习辅助平台。

可用于答辩或项目介绍的一句话概括：

> 系统围绕考研资料构建个人知识库，通过 RAG 问答、知识画像、定制练题、错题复盘和学习计划，把资料使用行为转化为可量化的学习反馈闭环。

## 2. 最值得突出的核心价值

### 2.1 从资料存储到学习闭环

项目不是单纯实现文件上传和问答，而是把多个学习环节串联起来：

1. 用户上传资料，系统抽取文本并生成知识片段。
2. 用户基于资料问答，系统返回可追溯来源。
3. 用户对来源反馈“很清楚 / 忘记了”，回写知识片段掌握情况。
4. 定制练题基于薄弱知识片段生成练习题。
5. 定制题或手动错题可关联知识片段。
6. 错题练习结果“写对了 / 写错了”继续回写知识画像。
7. 知识画像识别薄弱点、高风险点和复习压力。
8. 画像建议可写入学习计划，形成复习行动。

这条链路体现了系统的完整性：资料、问答、反馈、诊断、练习和计划不是割裂模块，而是围绕 `KnowledgeChunk` 形成数据闭环。

### 2.2 面向考研场景的个性化学习画像

系统以考研日期、学科文件夹、资料文件和知识片段为维度，统计覆盖率、掌握率、薄弱点、高风险点、学习趋势和复习压力。相比普通知识库问答系统，本项目突出“备考诊断”能力：

- 按一级学科文件夹聚合学科画像。
- 按资料文件聚合教材画像。
- 按知识片段反馈和错题练习计算掌握度。
- 根据最近练习时间、错误次数、引用次数和考研临近程度计算风险。
- 支持 7 天、14 天、30 天趋势分析。
- 支持规则诊断和可选 AI 总结。
- 支持将复习建议加入学习计划。

### 2.3 RAG 问答具有来源追溯和降级能力

知识问答采用混合检索增强生成方案。用户提问时，系统在当前文件夹及其子文件夹范围内检索知识片段，再把片段作为上下文交给大模型生成答案。

重点可突出：

- 支持关键词检索和向量检索。
- 使用 RRF 融合多路召回结果。
- 使用规则 rerank 和多样性控制，最终最多选取 5 个片段进入 Prompt。
- 支持 SSE 流式输出。
- 支持深度回答，通过查询改写补充检索。
- 返回答案来源，来源包含文件、页码、摘要和学习统计。
- Elasticsearch、Embedding 或大模型不可用时，系统可回退到数据库检索和本地摘要。

这说明系统兼顾了智能化效果、可解释性和本地演示稳定性。

### 2.4 知识切片从固定窗口升级为语义切片

系统将原有固定字符窗口切分改为“句子边界 + 标题识别 + 语义连续性合并 + 句子级重叠”的切片策略。

突出点：

- 避免从句子中间截断知识点。
- 标题、段落、自然停顿和主题转折词会影响切片边界。
- 目标长度约 800 字符，最大约 1100 字符，过短尾段自动合并。
- 相邻片段使用句子级重叠，而不是固定字符重叠。
- `chunkingVersion = 2`，可识别并重建旧版本切片。

该设计让每个 `KnowledgeChunk` 更接近一个完整知识点，既提高 RAG 检索质量，也让后续知识掌握统计更可靠。

### 2.5 定制练题和错题集形成复习闭环

定制练题不是普通聊天，而是固定使用知识库和来源追溯，每次选择一个知识片段出题。选题综合考虑相关性、掌握度、遗忘风险和引用关注度。

可重点说明：

- 定制题可追溯到具体知识片段。
- 用户可对定制题反馈“很清楚 / 忘记了”。
- 定制题可一键加入错题集并保留 chunk 关联。
- 手动错题也可按学科、资料、关键词关联知识片段。
- 刷错题结果会回写关联 chunk 的正确或错误次数。
- 没有关联 chunk 的错题也可正常练习，保证业务容错。

这体现了项目从“问答生成”走向“学习行为追踪”的设计深度。

## 3. 技术架构亮点

### 3.1 前后端分离架构

项目采用前后端分离模式：

- 前端：Vue 3、Vite、ECharts、lucide-vue-next。
- 后端：Spring Boot 3.3.5、Spring MVC、Spring Security、Spring Data JPA。
- 数据库：MySQL 8.x。
- 文本处理：PDFBox、Apache POI、Tesseract OCR。
- 检索增强：Elasticsearch、Embedding、OpenAI 兼容 Chat Completions 接口。

前端业务状态主要集中在 `frontend/src/composables/useSmartExamApp.js`，API 封装集中在 `frontend/src/api/client.js`。后端按 Controller、Service、Repository、Model、DTO 分层组织，业务边界清晰。

### 3.2 安全与用户隔离

系统支持注册登录，后端通过 JWT 鉴权，密码使用 BCrypt 哈希保存。所有业务资源都围绕当前登录用户进行归属校验，包括文件夹、资料文件、知识片段、错题、标签和学习计划。

重点可突出：

- 除注册登录外，业务 API 均需 Bearer Token。
- 服务层校验资源归属，防止用户越权访问。
- 文件夹、资料、错题、计划和知识片段均按用户隔离。
- 上传目录由配置控制，避免任意路径写入。

### 3.3 外部增强能力可降级

系统将 AI、Embedding、Elasticsearch 和 OCR 设计为增强能力，而不是基础功能的唯一依赖。

典型降级策略：

- 未配置 Embedding：跳过向量检索，保留关键词检索。
- Elasticsearch 不可用：回退数据库本地评分。
- 未配置聊天模型：知识库问答回退本地摘要。
- 模型接口异常：知识库场景返回兜底摘要。
- OCR 不可用：用户仍可手动补全文本后入库。

这种设计保证了毕业设计演示时系统可以稳定运行，也体现了工程可靠性。

## 5. 数据模型重点

建议答辩中重点讲以下实体关系：

- `User`：用户账号与登录身份。
- `UserStudyProfile`：考研日期、是否完成初始化、学科数量。
- `StudyFolder`：资料文件夹，一级文件夹作为学科维度。
- `StudyFile`：上传资料元数据、抽取文本和知识库状态。
- `KnowledgeChunk`：知识片段，是问答、画像、定制题和错题回写的核心。
- `KnowledgeChunkEvent`：记录引用、反馈和练习等学习事件。
- `MistakeQuestion`：错题主体。
- `MistakeQuestionChunk`：错题与知识片段关联。
- `MistakePracticeEvent`：错题复盘事件。
- `StudyPlanItem`：学习计划任务。
- `UserAiSettings`：用户级 AI 和 Embedding 配置。

其中 `KnowledgeChunk` 是最关键的数据中枢。系统通过它把资料内容、问答来源、学习反馈、错题练习和画像诊断连接起来。

## 6. 可作为创新点表述的内容

### 6.1 面向个人学习的闭环式 RAG

普通 RAG 系统通常只完成“检索资料并回答问题”。本项目进一步记录答案来源的引用次数和用户反馈，再把这些行为用于知识画像、定制练题和错题复盘，形成“检索问答 -> 反馈记录 -> 学习诊断 -> 复习行动”的闭环。

### 6.2 语义切片服务于问答和掌握统计

切片算法不只考虑检索召回，还考虑后续掌握度统计。通过句子边界和语义连续性生成知识片段，使每个片段更接近一个可学习、可反馈、可复习的知识点单元。

### 6.3 知识画像与考研倒计时结合

系统把考研日期纳入风险评估，使临近考试时复习压力更敏感。知识点风险不只由错题数量决定，还综合掌握度、遗忘间隔、引用关注度和考试紧迫度。

### 6.4 定制练题与错题集联动

定制练题生成的题目可保留来源 chunk，并直接进入错题集。刷题结果再回写知识片段，使 AI 出题不只是一次性生成内容，而是进入长期学习统计。

### 6.5 AI 计划采用预览后应用

AI 学习计划不会直接改写真实计划，而是先生成 CREATE、UPDATE、DELETE 操作草稿和预览结果，用户确认后再写入。这降低了 AI 误操作风险，体现了人机协同设计。

## 7. 答辩演示建议

推荐演示顺序：

1. 登录演示账号，展示首页、考研倒计时和今日计划。
2. 进入知识库，展示学科文件夹、资料列表和上传/编辑入口。
3. 打开知识问答，提问一个资料相关问题，展示流式回答和来源引用。
4. 点击来源，展示掌握度、引用次数和“很清楚 / 忘记了”反馈。
5. 切换定制练题，基于资料生成一道可追溯练习题。
6. 将定制题加入错题集，展示 chunk 关联。
7. 进入错题练习，记录“写对了 / 写错了”，说明会回写知识画像。
8. 打开知识画像，展示总览、学科、趋势、风险和诊断建议。
9. 将画像中的复习建议加入学习计划。
10. 打开学习计划，展示建议已经转化为可执行任务。

这条演示路线能完整证明系统的闭环能力。

## 10. 后续优化方向

可以在答辩中作为展望说明：

- 加强生产环境密钥管理，避免把真实 API Key 明文写入配置文件。
- 增加管理员、班级或教师端，实现多人协作学习管理。
- 引入对象存储和文件备份机制，提升生产环境可靠性。
- 增加更细粒度的知识点标签和章节结构识别。
- 增加自动化测试覆盖，特别是问答检索、权限隔离和错题回写链路。
- 对前端图表和大体积依赖进行按需加载，优化构建体积。
- 增加更多模型供应商预设和模型调用限流机制。

## 11. 答辩总结表述

本项目的重点不只是完成一个 Spring Boot + Vue 的前后端系统，而是在考研学习场景下，把资料管理、知识问答、学习画像、定制练题、错题复盘和学习计划整合成连续闭环。系统以 `KnowledgeChunk` 为核心数据单元，通过来源引用、用户反馈和练习结果持续更新掌握情况，再把这些数据用于薄弱点诊断和复习计划生成。整体设计兼顾了智能问答效果、来源可解释性、学习过程追踪、用户数据隔离和外部服务降级，具有较完整的工程实现和较明确的应用价值。
