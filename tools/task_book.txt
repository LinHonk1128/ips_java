毕 业 设 计（论 文）任 务 书
1．本毕业设计（论文）课题的目的和要求：
1.1目的：随着生成式人工智能与大语言模型技术的发展，传统在线学习系统已经难以满足考研学习场景下对个性化、智能化和知识关联能力的需求。现有学习平台普遍存在以下问题：资料管理分散、知识检索效率低、学习规划缺乏动态调整能力、错题复盘流程碎片化，以及问答系统缺少基于用户私有资料的上下文理解能力。传统学习平台主要采用“文件存储 + 关键词搜索”的交互模式，无法有效理解用户自然语言问题，也无法基于用户上传资料进行深层知识推理。当用户面对大量 PDF、Word、笔记及图片资料时，往往需要手动查阅和整理，学习效率较低。同时，传统问答系统缺乏知识增强机制，容易出现回答与学习资料脱节、内容泛化和上下文不一致等问题。本课题以“智能考研学习辅助”为核心目标，构建一个融合大语言模型（LLM）、检索增强生成（RAG）、知识库检索、学习计划管理与错题分析的智能考研系统，实现从“资料存储”向“知识理解与智能辅助”的学习模式转变。系统重点解决以下关键问题：用户学习资料分散，缺少统一知识管理能力；传统关键词检索无法满足复杂语义问答需求；学习计划静态化，无法根据学习情况动态调整；错题数据难以形成有效复盘与知识关联；AI 问答无法与用户私有学习资料建立深度联系。本设计通过引入 RAG 检索增强生成机制，实现“资料上传—文本抽取—知识切片—混合检索—大模型生成”的完整知识问答流程；同时结合学习计划、错题集和 AI 参数配置等模块，构建完整的智能学习闭环。1.2要求：按照软件工程规范完成系统需求分析、概要设计、详细设计、数据库设计、系统实现与测试；构建前后端分离架构，实现 Vue3 + Spring Boot 3 的现代 Web 系统；实现用户注册、登录与 JWT 身份认证机制，保证用户数据隔离与系统安全；实现学习资料上传、文本抽取、OCR 识别与知识库管理功能；构建基于 RAG 的知识增强问答系统，实现混合检索、上下文构造与来源引用；实现 Elasticsearch 与 Embedding 向量检索机制，提高语义检索能力；实现学习计划模块，包括手动计划管理与 AI 学习规划功能；实现错题集模块，包括错题录入、状态管理、随机练习与标签分类；完成 AI 设置模块，实现模型、API Key、Prompt 等参数配置；完成系统联调、异常处理与性能测试，确保系统稳定运行；编写完整毕业论文与相关技术文档，满足学院毕业设计规范要求。
本毕业设计（论文）课题的技术要求与数据（或论文主要内容）：
2.1电商环境关键问题凝练结合当前在线学习系统与考研辅助平台的实际情况，现有系统主要存在以下问题：（1）学习资料管理效率低考研用户通常需要管理大量教材、笔记、PDF、图片和历年真题。传统系统仅提供简单文件存储功能，缺少知识组织与智能检索能力，导致用户需要反复手动查阅资料。（2）传统搜索无法满足语义理解需求传统学习平台主要依赖关键词搜索，无法理解用户自然语言问题中的上下文语义。当用户提出综合性、概念性或推理型问题时，系统难以准确匹配相关知识内容。（3）AI 问答缺少知识增强能力普通大模型问答无法直接使用用户私有资料作为知识来源，容易产生“幻觉”问题，回答缺乏可信度，也无法定位具体资料出处。（4）学习规划缺少动态智能化能力传统学习计划功能以静态日程管理为主，无法结合学习情况进行智能推荐与动态调整，难以满足长期考研复习需求。（5）错题复盘体系不完善多数学习平台缺少完整的错题归档、状态跟踪与随机练习机制，用户难以形成系统性的复盘流程。（6）AI 与学习系统耦合程度低当前很多 AI 功能仅停留在简单聊天层面，没有真正融入学习资料、学习计划和错题体系，无法形成完整学习闭环。2.2核心研究内容（1）智能考研系统总体架构设计构建前后端分离的智能学习平台，实现知识库、问答系统、学习计划、错题集与 AI 设置等模块协同运行。系统采用：Vue3 + Vite 前端架构；Spring Boot 3 后端架构；RESTful API 通信；JWT 身份认证机制；H2/MySQL 数据存储；Elasticsearch 检索服务；OpenAI 兼容大模型接口。（2）基于 RAG 的知识增强问答机制系统核心研究内容为基于 RAG 的知识问答流程，包括：用户上传学习资料；系统自动进行文本抽取与 OCR；文本切分为 KnowledgeChunk；构建向量索引与关键词索引；执行混合检索；通过 rerank 选择高质量片段；构造 Prompt；调用大模型生成答案；返回来源引用。（3）知识库构建与混合检索机制系统采用“关键词检索 + 向量检索”混合模式。关键词检索基于 Elasticsearch multi_match； 向量检索基于 Embedding 与 dense_vector； 融合策略采用 RRF（Reciprocal Rank Fusion）。系统同时实现：查询改写；rerank 排序；多样性片段选择；Elasticsearch 降级；本地检索兜底。（4）学习计划智能辅助机制系统支持：周计划展示；手动新增、修改与删除计划；AI 学习规划建议；AI 草稿生成；用户确认后写入真实日程。AI 模块通过自然语言交互辅助用户进行复习规划，提高学习效率。（5）错题集与复盘机制系统支持：上传错题文本与附件；OCR 自动识别；错题状态管理；科目标签分类；随机练习；掌握程度跟踪。实现错题知识化与结构化管理。（6）AI 配置与模型管理机制系统允许用户自主配置：聊天模型；Embedding 模型；API Endpoint；API Key；系统 Prompt；AI 角色定位。以提高系统灵活性与扩展能力。2.3系统总体结构系统整体采用分层式前后端分离架构。表现层Vue3 Web 前端知识问答界面学习规划界面错题集界面AI 设置界面二、接口层RESTful APISSE 流式接口JWT 鉴权接口三、业务层用户认证模块文件管理模块文本抽取模块知识问答模块学习计划模块错题管理模块AI 设置模块四、数据层H2/MySQL：业务数据存储Elasticsearch：知识检索与向量索引uploads：本地文件存储2.4关键技术路线后端技术Spring Boot 3Spring MVCSpring SecurityJWTSpring Data JPAHibernateElasticsearch Java API前端技术Vue 3ViteJavaScriptAxiosSSE数据与检索技术H2/MySQLElasticsearchEmbedding 向量检索RRF 融合排序AI 与文本处理技术OpenAI Compatible APIRAG（检索增强生成）Prompt EngineeringApache POIPDFBoxTesseract OCR2.5创新点凝练（1）基于 RAG 的智能学习问答机制将用户上传资料转化为知识片段，通过混合检索与大模型生成，实现基于私有知识库的智能问答，提高回答准确性与可信度。（2）混合检索与多阶段优化机制系统结合关键词检索、向量检索、RRF 融合、rerank 与多样性控制，实现更高质量的知识召回效果。（3）深度回答与查询改写机制系统支持查询改写与二次检索，能够提升复杂问题、多章节问题与语义不一致问题的召回能力。（4）知识来源引用机制系统能够返回知识来源编号、文件名、页码与摘要，提高回答可解释性与学习可信度。（5）AI 与学习业务深度融合将 AI 与知识库、学习计划、错题集深度结合，形成完整学习辅助闭环，而非单纯聊天系统。（6）多级降级与本地兜底机制当 Elasticsearch、Embedding 或大模型不可用时，系统仍可通过本地检索与本地摘要继续运行，提高系统稳定性。
